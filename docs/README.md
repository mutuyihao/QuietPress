# Documentation

当前文档按用途分层：

- [setup.md](setup.md): 从零启动项目，包括 Supabase、环境变量、首次管理员、部署。
- [architecture.md](architecture.md): 当前代码结构、路由、数据模型、安全边界和已知限制。
- [release-checklist.md](release-checklist.md): 第一次发布到 GitHub 前后的检查清单。
- [project-audit.md](project-audit.md): 历史审计记录。
- [project-plan.md](project-plan.md): 历史问题整合和修复计划。

## 维护规则

- README 只保留项目概览、快速开始、核心链接和发布状态。
- setup 文档只写可执行步骤，不记录长期规划。
- architecture 文档必须和当前代码保持一致；功能变更时同步更新。
- release-checklist 记录发布门禁和首发流程。
- 历史 audit 文档不作为当前状态来源，除非重新审计后更新。
